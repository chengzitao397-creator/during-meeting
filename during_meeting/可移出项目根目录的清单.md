# 可移到另一个文件夹的清单

下面这些**不参与「网站 + 主流水线」运行**，可以整体挪到别的文件夹（例如 `文档/`、`脚本_辅助/`、`参考数据/`），减少根目录和推库时的文件量。挪动后若某脚本还要用，再在脚本里改路径或做符号链接即可。

---

## 一、建议移到「文档」或「报告」文件夹

| 文件 | 说明 |
|------|------|
| 流水线会议筛选原因_报告.md | 报告，由脚本生成，不参与运行 |
| 会议统计报告.md | 报告，由脚本生成 |

| 会议大全.md | 会议大全表格，参考用 |
| 会议时间轴交易日_使用说明_副本.txt | 使用说明 |

---

## 四、必须留在项目里的（不要移走）

- **根目录**  
  - `会议历届时间_副本.csv`（主流水线输入）  
  - `会议名单.csv`（统计/报告脚本可能用，可选留）  
  - `config_local.py.example`、`requirements.txt`、`.gitignore`
- **web/**  
  - 整个目录（网站代码 + 依赖说明）
- **计算逻辑/** 里主流水线（建议保留在根下「计算逻辑」）  
  - `会议窗口行业数据_米筐_中信30.py`  
  - `生成受影响显著一级行业_中信30.py`  
  - `生成受影响显著历届平均涨跌幅_中信30.py`  
  - `受影响显著一级行业_按月份分类_中信30.py`  
  - `按年按月份拆表_中信30.py`  
  - `生成冲突月权重_中信30.py`（若后续还用冲突月权重）
- **output/**  
  - 网站只读 `output/{年}/受影响显著一级行业_{年}M{月}_中信30.csv`（2022～2026 各 12 个月）  
  - 主流水线还会读：`会议窗口行业数据_中信30.csv`、`受影响显著的一级行业_中信30_最终版.csv`  
  - 若为省仓库体积，可只提交上述「网站 + 主流水线」用到的文件，其余 output 可放入 `.gitignore` 或移到本机别的目录（不提交）

---

## 五、可选：整个「时间」文件夹

- 若 `时间/` 里是历届时间表的另一套存放（如 会议历届时间.csv、不定期会议历届时间.csv），而主流水线只用根目录的 `会议历届时间_副本.csv`，则可将 `时间/` 整体移到「参考数据」或「归档」，需要时再拷回或改脚本路径。

---

**总结**：报告类 md/txt、参考用 csv、以及上面列出的辅助/归档脚本，都可以移到另一个文件夹；根目录和 `计算逻辑/` 里只保留主流水线 + 网站必需的文件，会清爽很多，也方便控制往 GitHub 推送的内容。
